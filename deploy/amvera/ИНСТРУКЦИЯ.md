# Публикация справочника и калькуляторов на Amvera

Сайт на Tilda **не меняется** — те же страницы:

| Страница Tilda | URL |
|----------------|-----|
| Справочник ПНАЭ | https://intech-atom.ru/pnae |
| Калькулятор (внутр.) | https://intech-atom.ru/calc1 |
| Калькулятор (наруж.) | https://intech-atom.ru/calc2 |

Меняется только **хостинг виджетов** (iframe): вместо Netlify — **Amvera Cloud**.

---

## 1. Подготовка пакетов на компьютере

```bash
# из корня репозитория
npm run publish:amvera
```

Будет выполнено:

1. `build:tilda` — сборка `tilda-hosted/`
2. `prepare:amvera` — копирование в `deploy/amvera/packages/` с Dockerfile и `amvera.yaml`

Папки для деплоя:

- `deploy/amvera/packages/01-spravochnik/`
- `deploy/amvera/packages/02-kalkulyator-vnutrennee-davlenie/`
- `deploy/amvera/packages/03-kalkulyator-naruzhnoe-davlenie/`

---

## 2. Три проекта в Amvera

Зайдите в [Amvera Cloud](https://cloud.amvera.ru/) и создайте **три отдельных проекта**.

Рекомендуемые имена (латиница, для URL):

| Проект Amvera | Содержимое папки |
|---------------|------------------|
| `intech-pnae` | `deploy/amvera/packages/01-spravochnik/` |
| `intech-calc1` | `deploy/amvera/packages/02-kalkulyator-vnutrennee-davlenie/` |
| `intech-calc2` | `deploy/amvera/packages/03-kalkulyator-naruzhnoe-davlenie/` |

### Деплой через Git (рекомендуется Amvera)

Для каждого проекта:

1. В Amvera: **Создать проект** → привязать Git или использовать встроенный репозиторий Amvera.
2. В папке пакета инициализируйте git и отправьте код:

```bash
cd deploy/amvera/packages/01-spravochnik
git init
git add .
git commit -m "Deploy handbook"
git remote add amvera <URL из панели Amvera>
git push amvera master
```

3. Amvera соберёт Docker-образ (nginx + статика) и запустит приложение.
4. Повторите для `02-…` и `03-…`.

### Бесплатный домен Amvera

В каждом проекте: **Настройки → Доменные имена → Добавить → Бесплатный домен Amvera**.

Формат URL: `https://<имя-проекта>.<ваш-логин>.amvera.io/`

Пример:

- `https://intech-pnae.ivanov.amvera.io/`
- `https://intech-calc1.ivanov.amvera.io/`
- `https://intech-calc2.ivanov.amvera.io/`

Проверьте в браузере: при **прямом** открытии должен показываться заглушка «откройте на сайте ИНТЕХ-АТОМ» — это нормально (защита embed).

## 3. Обновить ссылки iframe для Tilda

Откройте `deploy/amvera/urls.json` и подставьте **реальные** URL Amvera:

```json
{
  "01-spravochnik": "https://intech-pnae.ivanov.amvera.io/",
  "02-kalkulyator-vnutrennee-davlenie": "https://intech-calc1.ivanov.amvera.io/",
  "03-kalkulyator-naruzhnoe-davlenie": "https://intech-calc2.ivanov.amvera.io/"
}
```

Пересоберите вставки для Tilda:

```bash
npm run build:tilda:amvera
```

В каждой папке `tilda-hosted/<проект>/` обновится файл **`TILDA-ВСТАВКА.html`**.

---

## 4. Обновить блоки на Tilda

На страницах **без смены URL**:

| Страница | Что вставить |
|----------|--------------|
| `/pnae` | код из `tilda-hosted/01-spravochnik/TILDA-ВСТАВКА.html` |
| `/calc1` | код из `tilda-hosted/02-kalkulyator-vnutrennee-davlenie/TILDA-ВСТАВКА.html` |
| `/calc2` | код из `tilda-hosted/03-kalkulyator-naruzhnoe-davlenie/TILDA-ВСТАВКА.html` |

Блок: **T123 «HTML-код»** — замените старый iframe (Netlify) на новый (Amvera).

**Опубликуйте** страницы Tilda.

---

## 5. Обновление после изменений в коде

```bash
npm run publish:amvera
# затем git push в каждый из трёх проектов Amvera
npm run build:tilda:amvera   # если менялись только URL
```

---

## Netlify (старый способ)

Если нужно вернуть Netlify:

```bash
npm run build:tilda:netlify
```

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| 502 на Amvera | Убедитесь, что `containerPort: 80` в `amvera.yaml` и nginx слушает 80 |
| Пустая страница в iframe | Проверьте, что в zip/git есть `index.html`, `assets/`, `data/` |
| Заглушка вместо виджета на Tilda | Опубликуйте страницу Tilda; виджет работает только во iframe с intech-atom.ru |
| Старый Netlify в iframe | Обновите `urls.json`, пересоберите `build:tilda:amvera`, вставьте новый `TILDA-ВСТАВКА.html` |

Документация Amvera: https://docs.amvera.ru/
