# Публикация на GitLab Pages и вставка в Tilda

Один репозиторий GitLab → один сайт Pages с тремя виджетами:

| Виджет | Путь на Pages | Страница Tilda |
|--------|---------------|----------------|
| Справочник | `/pnae/` | https://intech-atom.ru/pnae |
| Калькулятор (внутр.) | `/calc1/` | https://intech-atom.ru/calc1 |
| Калькулятор (наруж.) | `/calc2/` | https://intech-atom.ru/calc2 |

---

## 1. Репозиторий на GitLab

1. Создайте проект на [gitlab.com](https://gitlab.com) (или свой GitLab).
2. Загрузите код репозитория `stresscalc` (ветка `main` или `master`).
3. Убедитесь, что в корне есть файл `.gitlab-ci.yml`.

---

## 2. Включить GitLab Pages

1. **Settings → General → Visibility** — для бесплатного Pages репозиторий должен быть **Public** (или Internal на GitLab.com с подпиской).
2. После push в основную ветку откроется pipeline **pages**.
3. **Deploy → Pages** — появится URL вида:

   `https://<логин-или-группа>.gitlab.io/<имя-репозитория>/`

Пример:

`https://ivanov.gitlab.io/stresscalc/`

---

## 3. Прописать URL в проекте

Откройте `deploy/gitlab/urls.json` и замените `USERNAME` и `PROJECT`:

```json
{
  "01-spravochnik": "https://ivanov.gitlab.io/stresscalc/pnae/",
  "02-kalkulyator-vnutrennee-davlenie": "https://ivanov.gitlab.io/stresscalc/calc1/",
  "03-kalkulyator-naruzhnoe-davlenie": "https://ivanov.gitlab.io/stresscalc/calc2/"
}
```

Локально пересоберите вставки для Tilda:

```bash
npm run build:tilda:gitlab
```

В `tilda-hosted/<проект>/TILDA-ВСТАВКА.html` появятся правильные ссылки.

Закоммитьте `urls.json` и сделайте push — pipeline обновит Pages.

---

## 4. Вставить код в Tilda

| Страница | Файл |
|----------|------|
| `/pnae` | `tilda-hosted/01-spravochnik/TILDA-ВСТАВКА.html` |
| `/calc1` | `tilda-hosted/02-kalkulyator-vnutrennee-davlenie/TILDA-ВСТАВКА.html` |
| `/calc2` | `tilda-hosted/03-kalkulyator-naruzhnoe-davlenie/TILDA-ВСТАВКА.html` |

Блок **T123 «HTML-код»** → вставить iframe → **Опубликовать** страницу.

---

## 5. Обновление после правок в коде

```bash
git add .
git commit -m "Обновление справочника"
git push
```

GitLab CI выполнит `npm run publish:gitlab` и обновит Pages (1–3 мин).

Если менялись только URL — достаточно `npm run build:tilda:gitlab` и обновить блоки на Tilda.

---

## Локальная проверка

```bash
npm run publish:gitlab
npx serve public
```

Прямое открытие покажет заглушку «только на сайте» — это нормально (защита embed).

---

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Pipeline pages не запускается | Push в ветку по умолчанию (`main`/`master`) |
| 404 на `/pnae/` | Дождитесь зелёного pipeline; проверьте артефакт `public/pnae/index.html` |
| Старый Netlify в iframe | Обновите `deploy/gitlab/urls.json`, `npm run build:tilda:gitlab`, Tilda |
| Заглушка на Tilda | Опубликуйте страницу Tilda; referrer должен быть intech-atom.ru |

Документация GitLab Pages: https://docs.gitlab.com/ee/user/project/pages/
